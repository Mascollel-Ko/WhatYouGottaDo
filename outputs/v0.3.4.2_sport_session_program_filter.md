# v0.3.4.2 Sport Session Program Filter

## Purpose

This hotfix prevents sport sessions and match records from entering generated training programs.

The program generator plans training exercises: strength work, accessory work, functional drills, conditioning, prehab, and low-to-moderate fatigue transfer work. It does not schedule actual sport participation such as badminton matches, soccer games, basketball games, tennis sessions, or imported sport-session aggregates.

## Data Split

New `Exercise` fields:

- `activityKind`
- `planningEligibility`

Main values:

- `TRAINING_EXERCISE` + `PROGRAM_SELECTABLE`: may be selected by `ProgramSkeletonGenerator`.
- `SPORT_SESSION` / `MATCH_RECORD`: actual external sport load.
- `FATIGUE_ONLY` / `ANALYSIS_ONLY` / `HIDDEN`: excluded from program generation.

The Room migration is additive:

- DB `6 -> 7`
- add `exercises.activityKind`
- add `exercises.planningEligibility`

Existing records, completed sets, plans, and user data are not deleted or rewritten.

## Program Generation Filter

`ProgramSkeletonGenerator` now builds its initial candidate pool with:

- `activityKind == TRAINING_EXERCISE`
- `planningEligibility == PROGRAM_SELECTABLE`

The filter runs before equipment filtering, user exclusion text, scoring, and periodization selection. If all available candidates are sport sessions, the generator returns no program items instead of falling back to sport records.

## Fatigue / Analysis Role

Sport sessions remain valid history.

They can still affect:

- readiness and fatigue load
- badminton transfer load history
- trend analysis
- recent-load signals

They must not create:

- generated preview items
- saved `TrainingProgramItem` rows
- program-apply planned `WorkoutEntry` / `WorkoutSet` rows

## UI Text

The program editor clarifies that:

- badminton support ratio controls badminton-transfer-supportive training work, not badminton match scheduling.
- sport:strength ratio controls sport-transfer accessory work versus general strength work, not actual sport-session scheduling.

## Tests

`ProgramSkeletonGeneratorTest` verifies:

- badminton match records are excluded from badminton-support programs.
- sport sessions are excluded even at high badminton transfer ratios.
- generation returns no items when only sport sessions are available.
- sport sessions can remain fatigue-eligible while not being program-selectable.
