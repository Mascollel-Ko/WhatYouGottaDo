# v0.3.5.4 ProgramBuilder Policy Tables

## Scope

v0.3.5.4 makes the v0.3.5.3 architecture split usable as policy data. It does not add the full ratio x goal x days hidden-template library and does not mutate canonical exercise metadata.

Runtime flow remains:

```text
request
  -> ProgramTemplateCatalog (existing slot skeleton)
  -> FatigueSlotPolicy
  -> SlotCapabilityResolver
  -> ProgramBuilder selection and prescription
  -> CoverageAccountingPolicy and validators
```

## Slot Categories

The slot representation includes strength anchors, strength support, badminton transfer, power/reactive, recovery/prehab, and accessory categories. Two future-facing slots were added without forcing them into templates:

- `FOREARM_GRIP_ELBOW_SUPPORT`: direct support component.
- `OVERHEAD_SMASH_SUPPORT`: derived umbrella only; it cannot receive direct full coverage from one exercise.

## Exposure Numeric Mapping

`ExposureTargetTable.numericTarget` maps each qualitative priority to a finite minimum, preferred, and maximum for a requested day count and window length.

Two-week defaults:

| Priority | 1-2 days min/pref/max | 3-4 days min/pref/max | 5-7 days min/pref/max |
|---|---|---|---|
| HIGH | 1/1/2 | 1/2/4 | 2/3/6 |
| MODERATE_HIGH | 0/1/2 | 1/2/4 | 1/3/5 |
| MODERATE | 0/1/2 | 0/2/3 | 1/2/4 |
| LOW_MODERATE | 0/0/1 | 0/1/2 | 0/2/3 |
| LOW | 0/0/1 | 0/1/2 | 0/1/2 |
| OPTIONAL | 0/0/1 | 0/0/1 | 0/0/2 |

Longer windows scale in two-week units. `OPTIONAL` never creates a minimum or preferred requirement. Every priority has a finite maximum, so higher training frequency cannot imply unlimited dominance.

## Coverage Accounting

| Capability | Credit |
|---|---:|
| Effective primary | 1.00 |
| Extra malformed primary | 0.50 |
| Secondary | 0.50 |
| Weak/fallback | 0.25 |
| Umbrella direct primary | 0.50 maximum |

Additional constraints:

- Only one deterministic primary can receive full credit.
- Total credit from one exercise is capped at `2.0` across requested slots.
- `OVERHEAD_SMASH_SUPPORT` is derived from component exposure.
- One component hit gives weak umbrella credit; two or more give partial credit; derived umbrella credit is never full.
- Component slots are scapular support, athletic overhead press, rotational chain, forearm/grip/elbow support, and trunk anti-rotation.

## Fatigue Slot Policy

The existing global `ProgramFatigueGate` remains authoritative. The slot table adds primary-capability restrictions without changing the fatigue model.

| Slot group | GREEN | YELLOW | ORANGE | RED |
|---|---|---|---|---|
| Strength anchor | Normal | Normal | Controlled | Low load only |
| Single-leg/decel | Normal | Controlled | Low load only | Low load only |
| Footwork/COD/reactive | Normal | Controlled | Low load only | Avoid |
| Athletic overhead press | Normal | Controlled | Low load only | Avoid |
| Rotational kinetic chain | Normal | Controlled | Low load only | Avoid |
| Trunk stability | Normal | Normal | Controlled | Low load only |
| Scapular support | Normal | Normal | Controlled | Low load only |
| Calf/ankle | Normal | Normal | Controlled | Low load only |
| Recovery/prehab | Normal | Normal | Normal | Normal |
| Rehab-like activation | Normal | Normal | Normal | Normal |
| Accessory | Normal | Normal | Controlled | Low load only |

`LOW_LOAD_ONLY` rejects high-stress, heavy-lower, high-impact, and high-intensity-COD candidates. `AVOID` permits only an already-classified recovery candidate. Existing ORANGE/RED hard-day and impact gates remain in place.

## Capability Resolution

Priority order:

1. Canonical runtime identity metadata.
2. Canonical supporting metadata.
3. Structured legacy metadata when canonical metadata is absent.
4. Explicit narrow exercise-name fallback when metadata is missing or incomplete.

The resolver emits one primary, up to two secondary capabilities, and one weak capability. It also emits source, confidence, and warnings. Name-only matches are weak and never become primary. Broad display tokens such as plain `PRESS` or `CORE` are not used as name fallback rules.

Explicit future recognition includes landmine/half-kneeling/DB/KB overhead press variants, ViPR rotation/chop/lift/shovel/scoop, cable chop/lift, rotational throw, landmine rotation, woodchop, band chop/lift, and controlled kettlebell halo.
