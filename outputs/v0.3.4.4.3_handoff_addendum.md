# v0.3.4.4.3 Handoff Addendum

- Ponytail was not used for this hotfix.
- Initial profile birth year now allows partial numeric input and validates only on save.
- Strength average RPE is numeric input; badminton RPE is removed from the UI and ignored by initial adaptation calculation.
- Recovery sliders use left=bad and right=good. Existing `currentFatigue`, `currentSoreness`, and `currentStress` values migrate through DB `11 -> 12`.
- Backup export now includes `profileRecoveryScaleDirection=HIGH_IS_GOOD`; import handles older backups by converting the old bad-is-high fatigue/soreness/stress fields.
- Fatigue pressure now treats zero residual and low baseline ratio as LOW, preventing `0.0x` or zero residual from showing HIGH.
- Performance degradation no longer escalates on a single small e1RM drop; combined or repeated signals are required for ELEVATED/HIGH.
- Added schema `12.json`, targeted unit tests, and an android-test migration compile check.
