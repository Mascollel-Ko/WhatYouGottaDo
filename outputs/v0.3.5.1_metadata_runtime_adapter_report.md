# v0.3.5.1 Metadata Runtime Adapter Report

## Decision

The normalized v0.3.5.0 taxonomy is treated as the canonical future metadata model. Legacy `movementPattern`, `movementCategory`, fatigue weights, and old badminton fields remain compatibility fallbacks only. No canonical value is down-converted into an old enum during intake.

The source used for vocabulary verification was `outputs/v0.3.5.0_pass3_movement_planning_sidecar.csv` (215 rows, zero duplicate `stableKey`, every row `safeForSeedMutation=NO`). No row classification was re-inferred from the current app taxonomy.

## Implementation

- Added `RuntimeExerciseMetadata`, `MetadataTokenField`, `RuntimeExerciseMetadataCatalog`, and `ExerciseMetadataAdapter`.
- Added canonical CSV intake with quoted-field support.
- Pipe fields split only on `|`, trim whitespace, remove duplicate/empty tokens, interpret blank/`NONE` as empty, and retain the original raw string.
- Unknown analysis tokens remain available as `PRESERVED_SAFE_NO_OP`.
- Unknown progress metrics remain available as `UNSUPPORTED_SAFE_NO_OP`; the raw value is retained.
- Added explicit behavior mapping for all 19 Pass 3 progress metric values.
- Exposed movement, planning, stress, transfer, and source fields in both analysis metadata models.
- Updated badminton transfer mapping to prefer canonical `DIRECT`/`SUPPORTIVE`/`GENERAL`/`NONE`, with legacy fallback.
- Updated program generation to prefer canonical planning eligibility, redundancy group, progress behavior, transfer level, source confidence, and stress magnitude when a runtime catalog is supplied.
- Added a stableKey-first runtime catalog resolver with exercise-name fallback.
- New backup set rows now include `stable_key`; restore resolves stableKey first and falls back to the exercise name for old backups.
- Updated app version to `v0.3.5.1` (`versionCode=305010`).

## Compatibility Boundary

The real seed files were not changed. Room entities and migrations were not changed. Therefore the current production app continues to run through legacy metadata unless an approved `RuntimeExerciseMetadataCatalog` is supplied. This is intentional: the patch establishes a lossless intake/runtime boundary before a separate seed/resource integration decision.

## Progress Metrics

Pass 3 values are mapped to broad behaviors without changing their raw values:

- Load/reps: `LOAD_REPS`, `LOAD_REPS_OR_REPS`, `LOAD_REPS_OR_TIME`, `MACHINE_LOAD_REPS`, `REPS_AT_LOAD`
- Volume load: `VOLUME_LOAD`
- Estimated 1RM: `ESTIMATED_1RM`
- Reps/time: `REPS_OR_TIME`, `TIME`, `TIME_OR_COMPLETION`, `TIME_OR_REPS`
- Distance/time: `DISTANCE_OR_TIME_LOAD`, `TIME_DISTANCE`, `TIME_DISTANCE_PACE_OR_INTENSITY`, `TIME_OR_DISTANCE`
- Session duration: `SESSION_DURATION`
- Quality: `QUALITY_BASED`, `QUALITY_LOAD_REPS`
- Count: `COUNT_ONLY`
- Future unknown: retained and excluded from unsupported calculations

## Analysis Tokens
