v0.3.5.0 Manual Review Judgement 반영 작업

목표:
첨부된 개별 판정표를 사용해 v0.3.5.0 seed metadata gap report의 manual review 판단을 정리하라.
이번 작업은 여전히 report/audit-only 작업이다.
seed, 앱 코드, DB schema, program generator는 수정하지 마라.

함께 제출되는 첨부 파일:
- v0.3.5.0_seed_metadata_gap_report.md
- v0.3.5.0_seed_metadata_gap_report.csv
- v0.3.5.0_manual_review_171_individual_judgement.csv
- v0.3.5.0_manual_review_171_individual_judgement.md

반드시 먼저 할 일:
1. 첨부 파일 4개를 확인하라.
2. 기존 gap report가 215개 exercise row를 읽고 215개 output CSV row를 만든 report-only 산출물인지 확인하라.
3. individual judgement CSV가 manualReviewRequired=YES였던 171개 row를 개별 판정한 파일인지 확인하라.
4. stableKey 또는 exerciseName 기준으로 gap report CSV와 judgement CSV를 매칭하라.
5. 매칭이 안 되는 row가 있으면 자동 추정하지 말고 audit report에 mismatch로 기록하라.

플래그 분리 규칙:
1. rowManualReviewRequired
   - 운동 family 자체가 틀렸거나 세부 subtype 판단이 필요한 경우 YES
   - P0/P1 family correction이 있으면 YES
   - 단순 source issue만 있으면 NO

2. sourceReviewRequired
   - sourceIdIssue가 MISSING_SOURCE_ID, NEEDS_NETWORK_RECHECK, UNKNOWN_SOURCE, DEPRECATED_SOURCE 등을 포함하면 YES
   - row family가 맞아도 source 검증이 필요하면 YES

3. templateMissingReviewRequired
   - FOREARM_GRIP_ACCESSORY_REVIEW
   - CHEST_ISOLATION_REVIEW
   - PULLOVER_LAT_CHEST_ACCESSORY_REVIEW
   - POWER_CORE_SHOULDER_REVIEW
   - RUNNING_MECHANICS_SUPPORT_REVIEW
   - BADMINTON_OVERHEAD_POWER_TEST_REVIEW
   - POSTERIOR_CHAIN_LOW_LOAD_REVIEW
   - OTHER_SPORT_SESSION_RECORDS_REVIEW
   같은 새 review family 또는 no-template family이면 YES

4. stressReviewRequired
   - HIGH_STRESS_SESSION_CLUSTER_REVIEW
   - HIGH_AXIAL_REVIEW
   - PLYOMETRIC_REVIEW
   - DECELERATION_REVIEW
   - ELASTIC_SSC_REVIEW
   또는 그와 동등한 high-stress reason이 있으면 YES
   - 단, v3에서 false positive를 줄인 상태를 유지하라.

5. sportSessionReviewRequired
   - BADMINTON_SESSION_SPORT_RECORDS
   - OTHER_SPORT_SESSION_RECORDS_REVIEW
   - sport session인데 PROGRAM_SELECTABLE 의심이 있는 row는 YES

6. sourceOnlyReview
   - sourceReviewRequired=YES
   - rowManualReviewRequired=NO
   - templateMissingReviewRequired=NO
   - stressReviewRequired=NO
   - sportSessionReviewRequired=NO
   인 경우 YES

7. overbroadManualReview
   - individual judgement에서 manual review가 과잉이라고 판정된 row는 YES
   - 또는 sourceOnlyReview=YES이면 YES

8. safeForTemplateDefault
   - family가 적절하고, source-only review 또는 low-risk template row이면 YES 가능
   - no-template family, sport session, badminton skill test, running mechanics review, high stress review, row subtype review는 NO 또는 PARTIAL

9. safeForSeedMutation
   - 이번 단계에서는 원칙적으로 NO
   - 단, 향후 후보로만 쓰려면 CANDIDATE 값을 허용할 수 있다.
   - 이 작업에서 실제 seed mutation은 절대 하지 마라.
