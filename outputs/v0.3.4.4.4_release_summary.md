# v0.3.4.4.4 Release Summary

## Version

- App version: `v0.3.4.4.4`
- Version code: `304044`
- Room DB version: `12`

## Main Purpose

v0.3.4.4.4 stabilizes the structured initial profile UX and fixes over-eager readiness labels. It also includes a UI follow-up for profile chips, record header layout, record exercise cards, and program list/detail usability.

## User-visible Changes

- Initial profile birth year accepts partial numeric input while typing and validates on save.
- Strength average RPE is numeric and supports one decimal place.
- Badminton RPE is no longer shown in the initial profile.
- Sleep quality, fatigue, soreness, stress, and condition use sliders.
- Main goal chips are easier to scan.
- Blank avoid-movement chips are filtered out.
- Record tab date navigation keeps the date centered and separates calendar access.
- Record exercise names get their own line; action buttons move below.
- Program cards expose an apply button directly.
- Program detail shows a plain day-by-day text summary.
- Program edit hides auto skeleton controls for existing programs.

## Analysis Changes

- Initial profile recovery slider direction is high-is-good.
- Badminton adaptation no longer depends on profile RPE.
- Zero residual fatigue or very low baseline ratio cannot produce a HIGH fatigue level by fallback.
- A single small e1RM dip does not produce ELEVATED performance degradation by itself.
