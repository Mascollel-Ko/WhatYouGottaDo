## Purpose

v0.3.4.5 is a focused usability and analysis wording patch. It keeps the v0.3.4.4.4 cold-start profile/readiness foundation and adds safer record deletion behavior, cleaner performance trend naming, corrected badminton training metric semantics, and a stricter rest overlay dismissal rule.

## User-Visible Changes

- Record tab exercise cards now provide an exercise-level delete action next to the detail action.
- Deleting the last remaining set from an unconfirmed exercise removes the exercise entry from today's record instead of leaving an undeletable shell.
- The rest timer overlay no longer appears when there is no next exercise target.
- The performance trend section title is shortened to `성과 추세`.
- Strength trend wording now uses `RPE 대비 운동량` instead of vague efficiency wording.
- Badminton trend wording now uses `셔틀 플레이 시간` for actual badminton play time.
- Detail metric chips are only shown in trend mode, where they are meaningful.
- Strength and fatigue ranking modes are removed from the detail selector; badminton ranking remains.
- Relationship analysis uses dropdown X/Y selectors and blocks selecting the same metric for both axes.

## Badminton Data Changes

- `배드민턴 레슨` is added to the exercise seed data.
- Existing direct badminton play items and the new lesson item use `BADMINTON_DIRECT_PLAY`.
- Direct play is counted as shuttle play time and is not mixed into footwork/reactive drill volume.
- Footwork drills, support exercises, and direct shuttle play are separated by structured metadata, not exercise-name parsing.

## Data Model Notes

- Room DB version remains `12`; this patch does not require a database migration.
- Exercise seed version is `6`, so the new badminton lesson metadata can be inserted through the existing seed path.
- `AnalysisExerciseFeatures` now carries stable identity fields used by the badminton metric calculator.
- APK and build outputs remain excluded from Git.
