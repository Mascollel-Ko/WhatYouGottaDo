# v0.3.5.0 Record-to-Metadata Mapping Plan

```text
Exercise seed / metadata sidecar
  -> ExerciseDefinition / ExerciseMetadata entity
  -> stableKey resolver
  -> TrainingRecord / SetRecord
  -> Analysis aggregation
  -> Fatigue score
  -> Muscle volume
  -> Badminton transfer
  -> Recommendation engine
```

| Step | Current status | Detail |
| --- | --- | --- |
| Exercise seed / metadata sidecar | METADATA_READY_BUT_NOT_USED | Final sidecar exists, but contains audit columns and normalization placeholders and is not an app asset. |
| ExerciseDefinition / ExerciseMetadata entity | PARTIAL; NEEDS_SCHEMA_CHANGE | Exercise entity exists with many metadata columns; newer family/subtype/slot/redundancy and split stress fields are missing or only approximated. |
| stableKey resolver | PARTIAL; NEEDS_REPOSITORY_LAYER | Unique stableKey and DAO lookup exist; one canonical record/import resolver does not. |
| TrainingRecord / SetRecord | PARTIAL | WorkoutEntry references exerciseId and name; Set references entryId. No stableKey snapshot. |
| Analysis aggregation | EXISTS | AnalysisInputCollector joins records to Exercise metadata by exerciseId. |
| Fatigue score | EXISTS; METADATA_READY_BUT_NOT_USED | Current structured fatigue metadata is used; final split stress taxonomy is not yet mapped. |
| Muscle volume | PARTIAL; METADATA_READY_BUT_NOT_USED | Primary/secondary muscle and hypertrophy groups exist; detailed contribution weights are not integrated. |
| Badminton transfer | EXISTS; METADATA_READY_BUT_NOT_USED | Existing transfer strength/roles are used; final level/type/qualities need an adapter. |
| Recommendation engine | PARTIAL; METADATA_READY_BUT_NOT_USED | Structured metadata drives current recommendations, but final programSlot/redundancyGroup and source confidence are not consumed. |

## Intended Post-v0.3.5.0 Flow

`normalized seed candidate stableKey -> Exercise row -> stableKey-first repository resolver -> WorkoutEntry.exerciseId -> ExerciseAnalysisMapper -> existing calculators/recommenders`

Schema change is needed only for final fields that cannot be represented safely by existing Exercise columns. A separate metadata table is optional, not automatically required.
