# v0.3.4.4.3 UI Follow-up Precheck Report

## Scope

This follow-up keeps `versionName = v0.3.4.4.3` and layers UI corrections on top of the existing v0.3.4.4.3 readiness hotfix work. Ponytail-related work was not performed.

## Current State

| Area | Current implementation | Issue | Handling in this patch |
| --- | --- | --- | --- |
| Initial profile goal chips | Goal options rendered in rows of three | Long labels can wrap awkwardly | Render goal chips two per row |
| Initial profile avoid movement chips | Multi-select renders every option label | Blank/invalid labels could render as empty chips | Filter blank option labels before rendering |
| Strength RPE | Numeric text field backed by `Double?`, but integer-only input | `7.5` cannot be entered | Allow one decimal place, validate `1.0..10.0` on save |
| Badminton RPE | Removed from UI in earlier v0.3.4.4.3 work | Must remain removed | No UI or adaptation dependency restored |
| Record exercise card | Exercise name and action buttons share one row | Long names truncate before action buttons | Exercise name gets its own first line; actions move to second line |
| Record date header | Previous/date/next/calendar all share one row | Calendar can push the date off center | Previous/date/next are one centered row; calendar is separated |
| Program list card | Apply action exists only inside program detail | User cannot apply directly from card | Add apply button to default card state |
| Program detail | Detail doubles as editing surface | Opening a program is too complex | Show day-by-day text summary only |
| Program edit | Auto skeleton generator controls visible during edit | Edit screen should focus existing program content | Hide generator controls when editing an existing program |
