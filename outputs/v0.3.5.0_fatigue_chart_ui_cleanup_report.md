# v0.3.5.0 Fatigue Chart UI Cleanup Report

## Source Selector

The fatigue target and grouping selectors remain visible. Only the third-level source selector is collapsed into `FatigueSourceMultiSelect`.

The selector supports:

- search by display label or source key
- multi-select with checkboxes
- clear all
- restore the top three contributors
- apply or close without applying
- compact collapsed summary such as `배드민턴 풋워크 외 2개`
- selected-period contribution percentage on every option row

Changing the fatigue target or grouping clears the user override, recalculates source options, and selects the new top three contributors. An explicitly applied empty selection remains empty and shows `표시할 항목을 선택하세요.`

## Chart Range

`FatigueAnalysisMapper` now finds the first date with confirmed training load and applies:

`chartStartDate = max(periodStartDate, firstWorkoutDate)`

Trend and contribution series are built from this clamped window. For 12-week and 28-week views, weekly aggregation happens only after clamping. Profile baseline seeding remains in the fatigue calculator but cannot create visible pre-record chart observations.

`TrainingRepository.homeTodaySummary()` applies the same display boundary to Home's seven-day training-load and fatigue sparklines. With no confirmed workout record, the mini-series are empty.
