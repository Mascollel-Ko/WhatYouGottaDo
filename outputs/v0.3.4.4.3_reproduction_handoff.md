## Current Data Model Notes

- Room DB version is `12`.
- v12 includes recovery scale direction migration for initial profile recovery values.
- Destructive migration is not allowed.
- Backup/restore must accept older profile fields defensively.
- `badmintonAverageRpe` may exist for compatibility but is not used by the current adaptation calculation.

## Initial Profile Rules

- Core calculation fields are structured: numeric, enum key, enum list, slider score, or boolean.
- Birth year is numeric, 4 digits, validated on save.
- Sex is one of `MALE`, `FEMALE`, `UNSPECIFIED`.
- Strength years and badminton years are separated.
- Strength average RPE is stored as `Double?` and accepts one decimal place.
- Badminton RPE is not shown or used.
- Pain and avoid movement fields store enum key lists.
- Recovery sliders are stored in the existing 1..5 fields with high-is-good direction.

## Readiness / Fatigue Logic

- Initial profile is converted into an adaptation profile.
- Cold-start readiness should be influenced by profile-derived resistance, activity, badminton, recovery, detraining, restriction, and goal sensitivity inputs.
- The profile must not act only as a `FATIGUED -> CAUTION` post-filter.
- Fatigue display must not show `0.0x` or residual `0` with HIGH unless another visible reason explains it.
- Missing baseline should be shown as insufficient data, not as `0.0x HIGH`.
- Single small e1RM dips should remain low/normal unless corroborated by other performance signals.

## Record Tab Current UI

- Date header:

## Program Tab Current UI

- Existing program card:
  - displays program name and metadata
  - exposes apply button directly
  - retains open, edit, delete
- Detail:
  - read-only day-by-day text summary
  - no inline editor
  - no apply card
- Edit:
  - existing program items are loaded into the existing skeleton editing path
  - auto skeleton controls are hidden for existing programs

## Backup / Restore Rules

- CSV daily timeseries import must not crash on empty or malformed numeric fields.
- `sleep_hours` is parsed defensively as numeric.
- Backup/export should store enum keys, not display labels, for structured profile values.
- Existing older backups should be accepted with safe fallbacks.
