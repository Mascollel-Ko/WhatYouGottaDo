## New UI Structure

### Simple mode

- Displays one OFI chart.
- Displays at most two highest current fatigue axes.
- Displays at most two lowest current fatigue axes as `여유 부하`.
- Does not display recommendation prose or six-axis detail.
- A zero-only history produces `기록이 부족합니다.` instead of a fake historical zero chart.

### Detail graph 1: fatigue trend

- Selectable series: OFI, neuromuscular, systemic muscular, local muscular, joint/tendon/impact, movement focus, and recovery pressure.
- Multiple series can be selected.
- Default selection is OFI.

### Detail graph 2: fatigue contribution trend

- Selects one fatigue target and displays source contribution rather than duplicating the six-axis score lines.
- Grouping options: `redundancyGroup`, `movementFamily`, `programSlot`, and `exerciseName`.
- Default grouping is `redundancyGroup`.
- The top three contributors are selected by default; up to eight source choices are exposed.
- Group contribution is produced by `DailyFatigueCalculator` decay output and mapped outside Compose.

### Period behavior

- Supports 2, 4, 8, 12, and 28 weeks.
- 2-8 weeks use daily points.
- 12 and 28 weeks use Monday-based weekly averages.

## Data Flow

`TrainingRepository.fatigueAnalysisHistory()` calculates up to 196 daily results on `Dispatchers.IO` using the existing `DailyFatigueCalculator`. `TrainingViewModel` stores selection state and invokes the pure `FatigueAnalysisMapper`. Compose receives final chart series and does not query the repository or calculate fatigue.
