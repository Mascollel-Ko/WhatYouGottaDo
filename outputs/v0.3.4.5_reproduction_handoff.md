## Record Deletion Rules

- Each exercise card in the record tab exposes a delete action near the detail action.
- Exercise-level delete removes the `WorkoutEntry` and all sets for that entry.
- Confirmed entries ask for confirmation before deletion.
- Unconfirmed entries can be removed without leaving an undeletable empty shell.
- Deleting the last set from an unconfirmed entry removes the entry itself.
- The implementation path is DAO -> repository -> view model -> `RecordScreen`.

## Rest Overlay Rules

The rest overlay must not remain after the app has no useful next exercise target. Visibility requires:

- timer is active,
- target record date is today,
- the timer started after a confirmed set,
- a next exercise target exists.

If there is no next exercise, the overlay is suppressed instead of forcing the user to dismiss it manually.

## Performance Trend Rules

First screen:

- Section label: `성과 추세`
- Exactly three summary chart concepts:
  - strength performance,
  - badminton training volume,
  - fatigue composite.
- The UI should emphasize trend, not a score-like judgment.

Detail screen:

- Strength explanation keeps trend, composition, and contribution modes.
- Strength ranking mode is removed.
- Badminton explanation keeps ranking mode because item ranking is useful and has clear structured metadata.
- Fatigue explanation keeps trend, composition, and contribution modes.
- Fatigue ranking mode is removed.
- Metric chips appear only in trend mode.
- Relationship analysis uses dropdown X/Y selectors.
- Selecting the same metric for X and Y is blocked.
- Relationship wording must describe correlation as a tendency, not causality.

## Current Metric Labels

- Strength efficiency wording is replaced by `RPE 대비 운동량`.
- Badminton court volume wording is replaced by `셔틀 플레이 시간`.
- Badminton training volume must not be described as skill or match performance.

## Badminton Metric Semantics

Badminton training volume is split by structured metadata:

- Direct shuttle play: exercises with `BADMINTON_DIRECT_PLAY`, including `배드민턴` and `배드민턴 레슨`.
- Footwork/reactive work: court movement and reactive drill metadata.
- Support work: transfer-support strength, grip, rotation, anti-rotation, overhead, SSC, and deceleration support metadata.

Do not classify badminton training by exercise-name string parsing.
