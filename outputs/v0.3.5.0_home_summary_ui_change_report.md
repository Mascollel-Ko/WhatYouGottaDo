# v0.3.5.0 Home Summary UI Changes

## Primary Actions

The Home primary CTA area now contains only:

- 오늘 운동 기록하기
- 프로그램으로 시작하기

`운동 목록 둘러보기` was removed only from the large Home CTA area. The exercise-list feature remains available through the persistent bottom `운동` tab.

The Home-specific action cards use 12dp vertical padding, 8dp inter-card spacing, `titleMedium`, `bodySmall`, and a 72dp minimum touch height. The shared `CtaCard` was not changed, avoiding layout changes on other screens.

## Today Summary

The compact dashboard now displays:

- fatigue label and OFI out of 100
- a short label-specific headline
- planned exercise count
- confirmed set count
- unconfirmed set count
- recent seven-day training-load sparkline
- recent seven-day OFI sparkline

The charts use real `MiniTrendPoint` values and intentionally omit legends, six-axis details, and trend interpretation. Full analysis UI was not changed.

## State Flow

`TrainingRepository.homeTodaySummary()` calculates the state off the main thread. `TrainingViewModel.homeTodaySummary` caches it as a `StateFlow`, and existing record/profile mutations refresh it through the current analysis refresh path.

Build and focused tests passed. No Room, seed, user DB, version, or Git history change was made.
