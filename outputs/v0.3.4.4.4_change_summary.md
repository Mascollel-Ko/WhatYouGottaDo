# v0.3.4.4.4 Change Summary

## Version

- `versionName`: `v0.3.4.4.4`

## Functional Changes

- Fixed birth-year entry so partial numeric input is allowed and final validation happens on save.
- Changed strength average RPE to numeric input with save-time `1.0..10.0` validation and one decimal place support.
- Removed badminton RPE from initial profile UI and adaptation calculations.
- Replaced five recovery chip fields with sliders using left=bad and right=good semantics.
- Migrated existing fatigue/soreness/stress profile values to the new higher-is-better direction.
- Exported a recovery scale direction key in backups and made restore compatible with old backups.
- Prevented zero residual or low baseline ratio from producing HIGH fatigue.
- Prevented a single small e1RM dip from producing ELEVATED performance degradation.
- Rendered main-goal profile chips two per row and filtered blank chip labels.
- Centered the record date between previous/next navigation and separated the calendar button.
- Moved record exercise action buttons below the exercise name.
- Added program apply entry points to default program cards.
- Changed program detail to a read-only day summary.
- Hid auto skeleton generation controls when editing an existing program.
