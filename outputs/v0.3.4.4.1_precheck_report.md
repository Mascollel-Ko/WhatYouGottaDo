# v0.3.4.4.1 Precheck Report

## 확인 결과

- `InitialProfileDialog.kt`는 v0.3.4.4에서 출생연도/성별/운동경력/통증/목표를 자유 텍스트로 입력받았다.
- `InitialUserProfile`은 `birthYearOrAgeRange`, `gender`, `strengthTrainingAge`, `badmintonTrainingAge`, `painAreas`, `avoidedMovements`, `goals` 같은 문자열 필드를 갖고 있었다.
- DB는 v9였고 `initial_user_profiles`는 v0.3.4.4에서 추가됐다.
- 초기 프로필은 `InitialProfileReadinessAdjuster`에서 cold-start `FATIGUED` 보정에 사용됐다.
- 백업/복원은 `row_type=profile`, `profile_key`, `profile_value` key/value 방식이다.

## 자유 텍스트 문제

- 성별, 출생연도, 경력, 통증 부위, 피하고 싶은 움직임, 목표가 분석용으로 안정적이지 않았다.
- 이번 hotfix는 기존 문자열 필드를 삭제하지 않고 deprecated/호환 필드로 남겼다.
- 새 분석 입력은 numeric/enum/tag 필드를 우선 사용한다.
