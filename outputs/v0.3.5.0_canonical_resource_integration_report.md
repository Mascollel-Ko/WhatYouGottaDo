## Resource Schema

The packaged asset contains identity, activity/planning, movement/program/progression, analysis eligibility, stress, transfer, source status, and the six Pass 3.1 stress axes. It intentionally excludes `proposed*`, `review*`, `proposal*`, `humanReview*`, `decisionPack*`, notes, seed mutation potential, and seed patch readiness fields.

The manifest records the source artifact and SHA-256, packaged asset SHA-256, row count, field list, distributions, and `safeForSeedMutation=NO` policy. The runtime asset does not duplicate review prose or per-workout metadata.

## Runtime Integration

- `RuntimeExerciseMetadataAssetLoader` validates the manifest hash, row count, and stableKey uniqueness.
- `RuntimeExerciseMetadataCatalogProvider` caches one catalog for repository reuse.
- `RuntimeExerciseMetadataCatalog` resolves stableKey first. Name fallback is explicit and restricted to custom or `imported_` legacy records.
- `TrainingRepository` injects the catalog into program generation, Analysis V3 input, badminton scoring, and today readiness.
- `AnalysisInputCollector` and `ExerciseAnalysisMapper` preserve all canonical stress and transfer fields, including the six axes.
- `DailyAnalysisLoadAggregator` applies the canonical neuromuscular, systemic, local, and joint/tendon axis levels as conservative modifiers over the existing load model.
- `ProgramSkeletonGenerator` prioritizes canonical planning eligibility, redundancy, progress behavior, transfer level, and stress magnitude.
- Restore uses stableKey first. Old non-custom backups without a stable key may resolve an exact canonical exercise name before creating an `imported_` key.

Pass 3.1 decisions remain unchanged, including one-arm cable row variants as SUPPORTIVE, Superman as GENERAL, leg extension/curl as LOW stress, and bench press/dips/bench dips as HIGH stress.
