# v0.3.8.0 Program Rationale Release Notes

v0.3.8.0 is a user-facing program UX improvement release.

## Changes

- Generated program preview items now show concise Korean reason labels.
- Reason labels are built from existing selected-exercise context such as anchor role, template slot, badminton transfer slot, fatigue placement, recovery/prehab role, and fallback source.
- The normal card view stays compact and shows a single `이유:` line.
- The ProgramBuilder selection algorithm, scoring, fatigue formulas, badminton transfer formulas, metadata taxonomy, seed data, and Room schema are unchanged.

## Examples

- `힌지 앵커 · 강약 배치 · 슬롯 조건 충족`
- `배드민턴 보조전이 · 방향전환 보조 · 감속 안정성`
- `회복성 보조 · 관절 부담 완화 · 피로도 고려`
- `동작계열 균형 · fallback 선택`

## Scope

- No excluded-candidate explanation UI.
- No detailed scoring breakdown.
- No new onboarding or help screen.
- No Room database migration.
- No release APK automation change.

## Verification

Focused unit tests cover anchor labels, badminton transfer/deceleration labels, fallback labels, red-fatigue recovery labels, and the four-label display cap.
