# v0.4.2.1 Release Notes

v0.4.2.1 is a badminton transfer analysis color distinction hotfix.

## Cause

Badminton transfer analysis could render different transfer categories with duplicated or visually indistinguishable colors. The method charts used a small shared chart palette through modulo indexing, and the transfer summary bars used one primary color for every category.

## Changes

- Added a fixed badminton transfer color palette keyed by transfer objective, axis, and transfer type.
- Added color keys to analysis bar and stacked-bar models without changing analysis values.
- Routed badminton transfer bars, stacked chart segments, and legend chips through the same category color mapper.
- Added regression tests for distinct colors, stable color mapping, shared legend/chart color keys, and preserved analysis values.
- Bumped the app version to `v0.4.2.1` / `402001`.

## Scope

- No ProgramBuilder changes.
- No fatigue formula changes.
- No badminton transfer calculation changes.
- No taxonomy/category changes.
- No Room schema changes.
- No backup/restore changes.
