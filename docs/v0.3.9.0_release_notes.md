# v0.3.9.0 Release Notes

Base:

```text
clean v0.3.6.4
```

Target:

```text
v0.3.9.0
```

Main theme:

```text
Consolidate fatigue presentation and session gating around the readiness-derived fatigue source.
```

## Completed Changes

- Residual fatigue persistence is now category-aware and RPE-aware.
- Body-part and tissue persistence was added for readiness fatigue pressure.
- Added shared `FatiguePresentationSnapshot` and `TrainingGateSnapshot`.
- Today Readiness exposes `fatiguePresentation`.
- Home fatigue card prefers readiness fatigue presentation and preserves legacy fallback.
- Analysis fatigue overview prefers readiness fatigue presentation and preserves legacy fallback.
- Program application uses `TrainingGateSnapshot` for today's newly inserted session items only.
- Current-day gate boundary is covered by focused tests.

## Known Limitation

`TrainingGateSnapshot.rpeCap` is not written into workout rows in this release.

Reason:

```text
TrainingProgramItem has no planned target RPE field.
WorkoutEntry.rpe and WorkoutSet.rpe are recorded effort fields, not planned target fields.
```

## Deferred To v0.3.9.1

Runtime adjustment of already-inserted planned rows is intentionally deferred.

Deferred scope:

```text
existing planned WorkoutEntry / WorkoutSet rows
only unconfirmed planned rows
preserve confirmed user data
do not modify recorded RPE
likely requires repository-level androidTest or an existing Room harness
```
