# v0.4.1.10.5 ProgramBuilder Reservoir Beam Worklog

## Baseline

- Started from latest `main` at `v0.4.1.10.4`.
- Working branch: `refactor/program-builder-reservoir-beam-v041105`.
- Goal: replace the remaining filter-cascade selection behavior with reservoir, slot-first composition, context-aware reranking, bounded top-k/beam choice, issue-driven repair, and evaluation score caps.

수정 골자:
- Added a focused reservoir/beam regression test file.
- Locked down that a missing-anchor, repeated-captain-chair style skeleton cannot earn a perfect score.
- Documented that 6-corner shadow footwork remains allowed as a badminton-transfer item when it is not a direct sport session.

파일별 변경:
- `ProgramBuilderReservoirBeamV041105Test.kt`: reservoir/beam regression fixtures.
- `docs/v0.4.1.10.5_program_builder_reservoir_beam_worklog.md`: release worklog started.

동작 변화:
- No production behavior change.

다음 작업자 참고:
- Future commits should activate actual reservoir/reranking/beam behavior against these failure modes.
- Do not "fix" the 6-corner case by removing it globally; the issue is foundation-anchor absence and filler repetition.

## Commit 2 - fix(program): derive week roles from periodization input

수정 원인:
- Periodization was selected late and did not provide explicit week roles/day profiles before slot filling.
- Reservoir and reranking decisions need periodization context before exercise selection starts.

수정 골자:
- Added periodization week role and day profile models.
- Added `ProgramPeriodizationPlanPolicy` for AUTO resolution, week plan adjustment, and week-specific schedule adaptation.
- Moved periodization selection to the beginning of `ProgramBuilder.build`.

파일별 변경:
- `ProgramPeriodizationModels.kt`: week role, day profile, and periodized week plan models.
- `ProgramPeriodizationPlanPolicy.kt`: periodization type resolution and schedule adaptation.
- `ProgramBuilder.kt`: resolves periodization before week planning and uses week-specific schedules.
- `ProgramBuilderReservoirBeamV041105Test.kt`: policy coverage for BADMINTON_WAVE, LINEAR_STRENGTH, and DAILY_UNDULATING behavior.

동작 변화:
- Existing periodization input now affects week roles and day profiles before exercise selection.
- No new user input was added.

다음 작업자 참고:
- Foundation reservation and context reranking should consume this periodization context instead of creating a second week-role system.

## Commit 3 - refactor(program): keep eligible candidates in a reservoir

수정 원인:
- The old filter cascade made it too easy for non-hard-gate policies to remove useful candidates before slot-level reranking.
- Foundation, loaded support, transfer, and core/accessory candidates need stable classification data that later policies can consume.

수정 골자:
- Added candidate tier, foundation pattern, and core pattern classification.
- Added `ProgramCandidateReservoir` to preserve hard-gate-eligible candidates and their classifications.
- Extended `ProgramCandidateInventoryResult` with a reservoir while keeping the existing `candidates` list for compatibility.

파일별 변경:
- `ProgramCandidateClassificationPolicy.kt`: classifies candidates into foundation, loaded support, transfer, core/accessory/prehab, or filler tiers.
- `ProgramCandidateReservoir.kt`: stores eligible candidates and classification lookup.
- `ProgramCandidateInventory.kt`: returns the reservoir with inventory results.
- `ProgramBuilderReservoirBeamV041105Test.kt`: verifies foundation and captain-chair/core classification.

동작 변화:
- No selection behavior change yet.
- Hard-gate-eligible candidates now have reservoir/classification metadata for later reranking and repair.

다음 작업자 참고:
- Repeat, fatigue, and variety should use this reservoir as scoring context, not as early destructive filters.

검증:
- `.\gradlew.bat :app:testDebugUnitTest --tests "*ProgramBuilder*"` passed.
- `.\gradlew.bat :app:compileDebugKotlin` passed.
