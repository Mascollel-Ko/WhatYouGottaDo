# v0.4.0.7 Record Mutation Extraction

This release does not add a new user-facing feature.

v0.4.0.7 is a behavior-preserving repository extraction release. It moves
record mutation/write responsibilities out of `TrainingRepository` without
changing record entry, selected copy, preserve-state selected copy, batch edit,
metadata, backup, analysis, plan, or UI behavior.

Key points:
- `TrainingRepository` remains the public entry point for record writes.
- `RecordMutationService` now owns workout entry and set mutation orchestration.
- Existing transaction boundaries and DAO call order are preserved.
- Confirmed/unconfirmed state meaning is unchanged.
- ViewModel, UI, backup, Room schema, metadata, and analysis formulas are unchanged.
