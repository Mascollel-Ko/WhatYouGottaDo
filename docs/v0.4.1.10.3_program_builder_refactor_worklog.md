- tight budget에서 optional slot을 줄이거나 skip할 수는 있지만, 전체 세션을 비우는 방향으로 돌아가면 안 된다.

## Commit 3 — refactor(program): extract scoring policy

Commit:
- pending

수정 원인:
- ProgramBuilder에 candidate score 계산, exposure balance, recovery/phase penalty가 남아 있었다.
- 점수 계산이 builder에 남으면 hard gate와 soft preference가 섞여 후보 붕괴 원인을 추적하기 어렵다.
- scoring은 탈락 판단이 아니라 ranking/preference 책임이므로 별도 policy로 분리해야 했다.

수정 골자:
- `ProgramScoringPolicy.kt`를 추가했다.
- `score()`와 `exposureBalanceAdjustment()`를 ProgramBuilder에서 ProgramScoringPolicy로 이동했다.
- performance week / recovery context penalty 상수도 scoring policy 내부로 이동했다.
- hard gate, fatigue gate, session gate, template gate는 ProgramBuilder/Query/기존 policy에 그대로 남겼다.
- 동작 변경 없는 scoring 책임 분리다.

파일별 변경:
- `ProgramScoringPolicy.kt`: candidate ranking 점수와 exposure balance 보정 책임.
- `ProgramBuilder.kt`: score 계산을 `scoringPolicy.score(...)`로 위임하고 scoring helper 제거.
- `docs/v0.4.1.10.3_program_builder_refactor_worklog.md`: Commit 3 기록.

다음 작업자 참고:
- ProgramScoringPolicy는 탈락을 결정하지 않는다. hard safety gate를 여기에 넣지 말 것.
- 후보가 부족하면 query trace/session constraint를 먼저 보고, 점수 policy는 ranking 품질을 볼 것.
- 다음 커밋은 adaptive pool과 controlled selection을 `ProgramVarietyPolicy`로 분리한다.
