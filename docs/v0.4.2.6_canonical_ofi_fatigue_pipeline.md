# v0.4.2.6 Canonical OFI Fatigue Pipeline Cleanup

## Baseline

- 기준: latest `origin/main`
- 시작 태그: `v0.4.2.6`
- 목적: Analysis 피로도 화면과 Home 피로도 카드가 canonical OFI 계산 결과를 단일 표시 소스로 사용하게 정리
- version/tag 변경: no

## Cause

피로도 화면에는 canonical OFI 계산 결과와 readiness presentation/detail section이 동시에 섞여 있었다. 이 때문에 `회복 지속` 같은 readiness residual 축이 사용자 표시 축, 그래프, 축별 경고, 기여 운동 분석에 노출될 수 있었고, 수행 감소/불편감 guardrail과 피로 축 판단의 의미도 화면에서 섞였다.

## Changes

- OFI 계산은 5개 canonical 축만 종합하도록 정리했다.
- 표시 가능한 피로 축은 `신경계`, `전신 근육`, `국소 근육`, `관절·건·충격`, `동작·집중`으로 제한했다.
- `회복 지속`은 사용자-facing 피로 축, 그래프, 축별 경고, 주요 기여 운동 축에서 제외했다.
- Analysis 피로도 화면의 현재 상태/판단 카드는 `DailyFatigueState` 기반 canonical OFI만 사용한다.
- `세부 판단 보기` readiness detail UI를 제거했다.
- `주의할 피로 축과 주요 기여 운동`은 canonical 축만 사용하고, 단일 축 문장은 축 이름을 한 번만 표시한다.
- `수행 감소 및 불편감 신호 정리` 섹션을 추가해 readiness safety guardrail을 피로 축 판단과 분리했다.
- `수면 보정 코칭 신호`는 별도 섹션으로 유지했다.
- Fatigue trend selector와 trend graph는 OFI + 5개 canonical 축만 노출한다.
- 이번 주 누적 부담은 fatigue analysis의 canonical OFI series를 사용한다.

## Preserved

- Readiness safety guardrail 자체는 유지했다.
- 통증/불편감, 수행 감소, 코트 회복 반응, jump/landing/deceleration/overhead/grip 관련 코칭 신호는 fatigue axis judgement와 분리해서 계속 표시한다.
- projected/expected fatigue 계산 흐름은 바꾸지 않았다.
- versionName/versionCode와 release tag는 변경하지 않았다.

## Follow-up UI Detail

- `주의할 피로 축과 주요 기여 운동` keeps its collapsed summary and adds a `자세히 보기` toggle.
- Expanded content always renders the canonical five axes as compact qualitative-state blocks; raw scores, thresholds, z-scores, and percentiles remain hidden.
- Only `높음` and `매우 높음` axes show exercise contributors. The first contributor is shown alone when its axis contribution is at least 1.5 times the second; otherwise the top two display names are shown.
- `수행 감소 및 불편감 신호` and `수면 보정 코칭 신호` are now grouped under one `인식 신호` card while preserving their separate data and empty-state messages.
- No OFI formula, threshold, readiness score, ProgramBuilder, record, backup, version, or tag behavior changes in this follow-up.
