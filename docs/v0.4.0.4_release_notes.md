# v0.4.0.4 PlanScreen UI Extraction

This release does not add a new user-facing feature.

It extracts `PlanScreen` UI sections into smaller composable files while
keeping plan screen state, validation, and ViewModel wiring in `PlanScreen.kt`.

Key points:
- `PlanScreen.kt` remains the owner of screen-level state, validation, and program generation/apply calls.
- Plan input controls and equipment options are split into a focused control file.
- Generated program preview UI is split into a focused preview file.
- Program cards and day summary/editor sections are split into focused section composables.
- Program generation, validation, metadata, backup, analysis, and record behavior are unchanged.
- ViewModel, Repository, ProgramBuilder, Room schema, metadata, backup, analysis, and record public APIs are unchanged.
