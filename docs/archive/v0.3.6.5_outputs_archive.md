# v0.3.6.5 Outputs Archive

This archive replaces the tracked `outputs/` tree removed during the v0.3.6.5 cleanup release.
The removed files were generated reports, validation queues, draft sidecars, manual-review handoff
inputs, and sample audit outputs. They were not app runtime inputs.

## Cleanup Decision

- Removed from source control: tracked generated files under `outputs/`.
- Added to `.gitignore`: `outputs/`, so local validation and report exports do not return as source files.
- Preserved as source of truth: app assets, Room schemas, Kotlin source, tests, and maintained docs.

## Metadata Audit Notes

The v0.3.5.0 metadata work produced many draft sidecars, review queues, source-verification reports,
template/stress/transfer reports, and normalization inventories. Those intermediate artifacts were
superseded by the canonical runtime metadata asset and its manifest:

- `app/src/main/assets/metadata/canonical_exercise_metadata_v0_3_5_0_pass3_1.csv`
- `app/src/main/assets/metadata/canonical_exercise_metadata_manifest.json`

Related maintained references remain in `docs/metadata_*` and
`docs/runtime_metadata_user_exercises_v0.3.5.1.md`.

## Runtime Metadata Persistence Notes

Runtime metadata persistence is guarded by source and test files rather than generated output reports.
The Room schema remains under `app/schemas/`, and v0.3.6.5 does not change the database schema.

Kept:

- `app/schemas/com.training.trackplanner.data.TrainingDatabase/`
- runtime metadata entity, mapper, DAO, repository, loader, and schema tests

## Program Builder Validation Notes

The v0.3.5.2 through v0.3.5.10 generated program reports and sample exports were audit artifacts.
Durable behavior now lives in ProgramBuilder source, policy tables, hidden template catalog code, and
focused tests.

Removed generated examples included representative template samples, generated program quality audit
reports, metadata slot coverage diagnostics, and fallback-role cleanup reports.

## Taxonomy Recalibration Notes

Deprecated taxonomy recalibration sidecars, blocked queues, source queues, decision logs, and draft
seed-candidate previews were removed from source control. Current app behavior should use canonical
metadata assets and runtime metadata code, not historical sidecar drafts.

## Backup / Restore Findings

Generated backup/restore and data migration reports under `outputs/` were removed. The actual
backup/restore implementation and tests remain in the app source tree.
