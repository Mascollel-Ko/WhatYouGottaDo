# v0.4.2.11 Release Notes

v0.4.2.11 adds an in-app first-user explanation flow without changing any
analysis, training, persistence, or calculation behavior.

## Baseline

- Started from local and remote `main` SHA
  `497e5a00f2bc7116fcb4664894ecd0d8182e6311`.
- Previous app identity was `versionName v0.4.2.10`,
  `versionCode 402010`.
- Six pre-existing user-owned `outputs/*` modifications remained untouched,
  unstaged, and uncommitted.

## Home Entry

- Added exactly one transparent compact row immediately below
  `프로그램으로 시작하기`.
- Exact label: `이 앱이 분석하는 것 보기`.
- The row has an information icon, trailing chevron, 48dp minimum touch
  target, and no subtitle, card, banner, badge, or automatic modal.
- It is outside the initial-profile card and remains available regardless of
  profile completion.

## Explanation Flow

- `app_explanation` introduces the product with the headline
  `운동 기록을 분석과 계획으로`, six concise feature rows, and one explicit
  limitation card.
- `analysis_guide` explains overall fatigue, five fatigue axes, local muscles,
  connective tissue, training distribution, practical high-score
  interpretation, and limited early calibration.
- `calculation_principles` distinguishes research from product policy,
  explains transparency and medical limitations, lists six protocol families,
  and opens the public protocol index in the system browser.
- Public protocol URL:
  `https://github.com/Mascollel-Ko/WhatYouGottaDo/tree/main/docs/protocols`.
- Missing browser handling shows an in-app message instead of crashing.
- System back returns guide and principles screens to the explanation screen,
  then returns to Home.

## Content And Accessibility

- All new visible Korean copy is stored in Android string resources.
- Interactive rows use at least 48dp touch targets with explicit content
  descriptions; decorative icons are not announced separately.
- Headings use Compose heading semantics and every screen is vertically
  scrollable.
- Deterministic Compose tests cover 360dp, 320dp, font scale 1.3, light and
  dark themes. Titles remain single-line at 360dp and content remains
  reachable at the compact accessible layout.
- The repository has no existing screenshot/golden framework, so this release
  reuses Robolectric Compose layout tests rather than adding a large new
  screenshot system.
