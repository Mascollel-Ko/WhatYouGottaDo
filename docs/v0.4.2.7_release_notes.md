# v0.4.2.7 Release Notes

v0.4.2.7 completes RCV-ALL-0.6 Phases 2 through 5. It imports the reviewed
unsided connective-tissue authority data, evaluates independent recovery
events, aggregates and ranks current state, and adds a dedicated Analysis
surface plus a summary-only OFI extension.

## Baseline and authority

- Continued from Phase 1 commit
  `b23c6ed3dd36dc7d6e4fd0de3075a2c517c130de`.
- Exercise/load authority:
  `REL-MSCP-D-INTEGRATED-0.3_UNSIDED`, SHA-256
  `efa3f0c47c4f5bf0ae634ed7e8656162ac6552b7a86659da28096cc257c50144`.
- DI/recovery authority: `RCV-ALL-0.6`, SHA-256
  `beb599e6a53fec92f999d1174bbb35ed31092aff56ad90ee0149e61dd4c615c9`.
- Generated assets cover 239 exercises, 3,507 authoritative load rows,
  50 protocol classes, 13 DI profiles, 21 recovery curves with 114 knots,
  7 routing rows, 15 joint complexes, and 77 unsided load units.

## Changes

### Phase 2 - deterministic assets

- Added a deterministic OpenXML generator and 16 typed CSV assets.
- Added typed parsing, validation, semantic counts, source hashes, and
  deterministic asset hashes.
- Preserved exactly one unresolved generic exercise rather than inventing a
  mapping.

### Phase 3 - recovery engine

- Added nullable exact `performedAt` persistence for newly confirmed records
  with Room schema version 19 and migration 18 to 19.
- Legacy date-only records remain date-only and are evaluated as conservative
  time ranges; no fake midnight or noon timestamp is assigned.
- Added immutable derived exposure events and independent recovery clocks.
- Added shape-preserving PCHIP interpolation with exact knots, bounded
  no-overshoot behavior, monotone preservation, and delayed-peak support.
- Calendar copies explicitly clear `performedAt`; copied planned records do
  not acquire invented observation times.

### Phase 4 - aggregation and ranking

- Aggregates residuals per unsided load-unit and load-dimension key.
- Prevents GROUP/component duplicate event IDs from inflating debt.
- Uses worst/max child state for joint summaries rather than summing children.
- Adds deterministic ranking, the exact 1.5 contributor rule, 56-day personal
  calibration, and immediate symptom override.

### Phase 5 - Analysis and OFI

- Added Analysis > Connective Tissue after fatigue, badminton, and strength.
- Shows all 15 joint complexes and all 77 unsided load units.
- Shows status, score, highest child, high-child count, current residual range,
  recent 56-day range, contributors, evidence, timestamp precision,
  calibration, and symptom diagnostics.
- Added the worst connective-tissue status, top three joint complexes, and a
  deep link to the fatigue card.
- Connective-tissue values are not added numerically to legacy OFI.

## Behavior preserved

- Legacy OFI numeric calculation, five axes, canonical wording, and projected
  fatigue behavior are unchanged.
- Readiness and ProgramBuilder behavior are unchanged.
- Backup/export/import behavior is unchanged apart from Room's nullable,
  backward-compatible `performedAt` field.
- Tissue state, recovery clocks, rankings, and UI remain unsided.
- Recovery values are modeled load/recovery state, not injury probability or
  diagnosis.

## Remaining risks

- Exact performed time exists only for newly confirmed records; legacy rows
  continue to use conservative date-only ranges.
- The symptom input currently applies a global joint/tendon discomfort
  override because the app does not collect per-tissue symptom locations.
- Personal normalized scores remain `CALIBRATING` until 56 observation days
  are available.
