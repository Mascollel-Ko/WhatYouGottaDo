# v0.5.0.1 Release Notes

v0.5.0.1 adds an experimental, metadata-constrained proxy performance model for bench press, squat and deadlift. It keeps confirmed actual e1RM unchanged while providing a separate estimate of major-lift capability, historical uncertainty and pre-session expected-versus-actual context.

## Why This Was Needed

The strength analysis showed only direct confirmed e1RM observations. A user who performed related presses, squat variants or hinge variants between direct major-lift sessions could not inspect whether those sessions provided evidence consistent with a change in the target lift. Simply inserting estimates into the actual series would have corrupted measurement semantics and introduced leakage risk.

The release therefore adds a separate derived-state subsystem with explicit provenance, uncertainty, rolling model selection and numerical failure guards.

## Calculation Contract

- Canonical actual e1RM remains the existing confirmed-set Epley maximum and is not recalculated by the posterior model.
- Only confirmed sets with 1~12 repetitions and positive effective load are eligible.
- One workout entry creates at most one session observation through a quality-weighted median of usable sets.
- Set RPE has priority over entry RPE. RIR is `clamp(10 - RPE, 0, 4)`.
- Missing RPE does not fabricate effort; it retains the observed load signal with higher variance.
- Bodyweight exercises reuse `BodyweightEffectiveLoadCalculator`, the latest valid bodyweight on or before the session, then the initial profile fallback.
- Exercise normalization is prior-only and stable-key specific, using log median/MAD over at most 64 previous observations.
- The fixed latent state contains exactly eight factors: shared press, horizontal press, knee extension, posterior-chain hip extension, trunk bracing, and bench/squat/deadlift specific residuals.
- Metadata loading coefficients are bounded product heuristics multiplied by effective metadata confidence. Unresolved custom exercises do not gain cross-loading through name matching.
- Daily state transition and Joseph-form covariance update use Apache Commons Math. Non-finite input, singular innovation covariance and failed positive-definite checks fail closed; bounded jitter is diagnostic and limited.
- Target-specific rolling backtests compare M0 LOCF, M1 target-only, M2 shared factors and M3 shared plus target-specific residuals. Cross-loading is selected only when prior-only error, interval width, log predictive density and minimum evidence gates support it.
- Model version and deterministic fingerprint: `proxy-performance-1.0.0`.

## User-Facing Changes

- Strength analysis now separates `실제 e1RM`, `추정 수행 상태`, `80% 추정 범위`, `세션 전 예상` and `예상 대비 수행`.
- Actual observations remain unconnected points. Posterior intervals use a historical interval band rather than the future `forecastRange` contract.
- Proxy-only weeks are visibly marked and are never presented as completed target-lift sessions.
- A separate experimental Lab card exposes selected model, rolling backtest metrics, loading evidence, recent innovations and numerical diagnostics.
- The UI states that the estimate does not replace actual records and is separate from the existing Bayesian time-series analysis.

## Preserved Boundaries

- No posterior value is added to `metricSeries` or the canonical e1RM series.
- `LegacyTimeSeriesAnalyzer` is unchanged and does not consume proxy posterior output.
- Strict time-series ingestion, BVAR/BLP preparation, lifecycle cells, transformation, scaling and shock contracts do not consume proxy posterior output.
- Workout records, confirmed/unconfirmed semantics, backup/restore, Room schema, OFI, connective-tissue analysis, badminton analysis, ProgramBuilder, home and readiness behavior are unchanged.
- Existing metadata authority and stable keys are reused; no exercise metadata asset was changed.

## Known Limits

- Loading coefficients and selection thresholds are bounded product heuristics, not user-calibrated causal effects.
- Without a prior direct target-lift history, the model does not invent an absolute kg baseline from proxy exercises alone.
- Self-reported RPE and metadata quality affect uncertainty and whether cross-loading is available.
- The 80% interval describes model uncertainty and is not a performance guarantee.
- The deterministic posterior does not preserve draws and is not suitable as a strict BVAR/BLP shock source.
