# v0.4.2.13 Release Notes

v0.4.2.13 rewrites every connective-tissue educational entry in ordinary Korean and simplifies the existing information dialog without changing connective-tissue calculations or any other production behavior.

## Baseline and identity

- Started from latest `origin/main` commit `7e2431539aa257ed0b89bf79c836ea80ec9f9942`, tagged `v0.4.2.12`.
- Previous identity: `v0.4.2.12 / 402012`.
- Release identity: `v0.4.2.13 / 402013`.
- Educational metadata: `RCV-ALL-0.6-EDU-2`.

## Why this changed

The former information copy frequently exposed internal anatomy and load-model terminology, repeated the selected name and generic description, and did not consistently tell a non-specialist where a structure is, what it does, or when exercise load on it becomes larger.

## Copy rewrite

- Rewrote all 77 `LOAD_UNIT` children individually before writing any parent copy.
- Synthesized all 15 `JOINT_COMPLEX` parents from their actual mapped children after the child review was complete.
- Preserved all 92 stable keys and the 77-to-15 parent hierarchy.
- Used practical body landmarks, tissue-appropriate function verbs, representative mapped movements, and structure-specific load-increasing conditions.
- Kept `shortDescriptionKo` for schema compatibility but removed it from the user-facing dialog; the final authority leaves it blank.
- Used no explicit injury-mechanism claim in this release. All 92 entries use conservative relative-load wording and do not equate a high app state with injury.
- Final educational CSV SHA-256: `c7b458ec4f84ef9cd6603d5ee786011358ebe4f535be7d9d91bccf56738f6d1b`.

## Information dialog

The existing parent and child information buttons still open one dialog. The selected structure remains the title. The body now contains exactly:

1. `위치`
2. `주요 기능`
3. `주로 사용되는 동작`

The existing general-information disclaimer and close action remain. Duplicate `표시명` and `설명` fields were removed, complete sentences no longer use middle-dot joins, and long content scrolls at normal and large accessibility font sizes. No new screen, image, binary asset, dependency, or second detail flow was added.

## Validation and review

- The existing asset validator enforces EDU-2 consistency, exact 92-key coverage, canonical display names, complete fields, natural sentence endings, a documented 1,000-character hard field limit, obsolete-boilerplate rejection, and exact complete-entry uniqueness.
