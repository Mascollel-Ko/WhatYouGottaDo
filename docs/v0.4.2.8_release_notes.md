# v0.4.2.8 Release Notes

v0.4.2.8 separates connective-tissue presentation from the five-axis OFI
summary and adds complete, structured educational information for the existing
unsided connective-tissue catalogue. The RCV-ALL-0.6 calculation engine is
unchanged.

## Baseline

- Started from verified `origin/main` commit
  `9b2bb2bc257b44108f24a1208e6eb04b58a4684c` (`v0.4.2.7`).
- Preserved the six pre-existing user-owned `outputs/*` modifications without
  editing, staging, or committing them.

## Changes

### Educational metadata

- Added one generated and validated production asset containing educational
  information for all 15 joint complexes and all 77 unsided load units.
- Each entry contains its stable key, scope, Korean display name, anatomical
  location, primary functions, common load contexts, optional short
  description, and metadata version.
- Validation fails on missing, blank, duplicate, unknown, or side-specific
  catalogue entries.
- The educational asset SHA-256 is
  `8089265e3e436179f4bace5dbf7edd0e1ac4ace339e975fa7d3e41b5786d460e`.

### OFI separation

- Removed connective-tissue status from the card that presents the current OFI
  state and judgment.
- Added a separate connective-tissue analysis navigation card below the
  existing OFI cards.
- The existing OFI contribution card remains limited to the five established
  fatigue axes.

## Behavior Preserved

- OFI still uses exactly the existing five axes, numeric calculation,
  thresholds, canonical classifier, labels, Korean messages, and axis counts.
- Connective-tissue values are not added to or averaged into OFI.
- RCV-ALL-0.6 event derivation, PCHIP recovery, 56-day calibration, unsided
  identities, aggregation, ranking, and deterministic tie-breaking are
  unchanged.
- Readiness, ProgramBuilder, record, backup, and import/export behavior are
  unchanged.

## Remaining Limitations

- Educational copy is general load-context information, not diagnosis,
  treatment advice, an injury-risk estimate, or a safe-load limit.
- Legacy records without exact performance time continue to use the existing
  conservative date-only recovery range.
- The current symptom input remains global rather than tissue-specific.
