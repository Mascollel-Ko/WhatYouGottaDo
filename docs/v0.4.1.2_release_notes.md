# v0.4.1.2 Calendar Record Service Extraction Release Notes

v0.4.1.2 is a behavior-preserving repository extraction release.
It moves low-risk calendar record delete/copy/move/range-copy orchestration out of `TrainingRepository` without changing confirmed/unconfirmed handling, overwrite behavior, completed timestamps, display order, backup, record mutation, program planning, metadata, analysis, home, readiness, or UI behavior.

## Changes

- Added `CalendarRecordService`.
- Delegated calendar conflict summary, date delete, date range delete, date copy, date move, and range copy-as-plan from `TrainingRepository`.
- Kept `TrainingRepository` public APIs and ViewModel/UI call sites unchanged.
- Left `ProgramPlanService`, `RecordMutationService`, backup/restore, DAO/schema, metadata, analysis, home, readiness, and UI paths untouched.
