# v0.4.1.10.7 Fatigue Threshold Relaxation Release Notes

## Start Baseline

- Started from latest main at `v0.4.1.10.6`.
- Baseline commit: `6ca0030`.
- Release version: `versionName = "v0.4.1.10.7"`, `versionCode = 401017`.

## Cause

Fatigue warnings and ProgramBuilder fatigue planning were still too sensitive. Normal post-workout load increases could appear as current fatigue warnings, and ProgramBuilder planning could downscale or block useful work before RED-level fatigue.

## Changes

- Added regression coverage for relaxed fatigue thresholds.
- Raised fatigue warning/caution/high thresholds by 15 percent across OFI labels, ProgramBuilder planning gates, readiness pressure, and presentation cutoffs.
- Moved ProgramBuilder planning downscale/block behavior to RED-level fatigue only.
- Softened Home projected-load wording so planned post-workout load does not read like current fatigue.
- Added Analysis projected OFI note text that distinguishes expected load spikes from unrecovered fatigue.
- Reworded phase-aware readiness and the Analysis readiness card to separate pre-workout/current fatigue from post-workout projected load.

## Behavior

- Current/pre-workout readiness can remain conservative.
- Post-workout projected load is treated as an expected training response unless it crosses the relaxed caution/high thresholds or shows carryover/repeated risk.
- ProgramBuilder planning keeps foundation anchors and normal prescriptions at YELLOW/ORANGE.
- RED planning still reduces high-risk work and blocks high-impact/reactive explosive planning candidates.
