# v0.3.7.0 Backup Restore Preflight Release Notes

v0.3.7.0 is a focused backup/restore reliability release.

## Audit Finding

The existing CSV backup and restore path already handled full restore CSVs, daily timeseries CSVs, daily check-ins, canonical sleep export, duplicate restored workout entries, and invalid-date row skipping.

The missing reliability layer was restore preflight:

- no user-visible restore preview before DB writes
- no date range or row-count summary before restore
- no explicit warning for suspicious `sleep_hours` / `body_weight_kg` values
- no warning for likely sleep/body-weight column swaps
- no warning for many unconfirmed planned sets with zero weight
- no blocking state for structurally invalid restore files

## Changes

- Adds restore preflight summary before import.
- Shows file type, date range, workout entry count, set count, daily metric count, check-in count, warning count, and blocking issue count.
- Blocks restore when required structure is missing, such as a missing `date` column.
- Warns for invalid dates, unknown row types, suspicious sleep/body-weight values, likely sleep/body-weight swaps, blank exercise identifiers, duplicate daily/check-in dates, and high shares of unconfirmed planned zero-weight sets.
- Keeps valid legacy restore and daily-timeseries CSV import behavior compatible.
- Keeps the actual import parser and Room schema unchanged.

## Warning Criteria

- `sleep_hours < 0` or `sleep_hours > 24`
- `body_weight_kg < 25` or `body_weight_kg > 250`
- `sleep_hours` looks like body weight while `body_weight_kg` looks like sleep
- invalid date rows
- unknown restore row type
- blank exercise name and blank stable key on set rows
- duplicate daily or check-in date rows
- at least five unconfirmed planned set rows where at least half have zero weight

## Blocking Criteria

- empty CSV file
- missing required `date` column

## Scope

- No Room schema change.
- No training, fatigue, metadata taxonomy, or ProgramBuilder logic change.
- No destructive restore behavior added.
- No aggressive auto-correction of suspicious values; the app warns before import.

## Verification

Focused backup/restore tests cover valid preflight summaries, missing required columns, invalid dates, likely sleep/body-weight swaps, and unconfirmed planned zero-weight warnings.
