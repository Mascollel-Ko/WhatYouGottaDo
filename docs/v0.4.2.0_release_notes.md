# v0.4.2.0 Release Notes

v0.4.2.0 starts the ProgramBuilder input and UI cleanup line.

## Cause

Several ProgramBuilder controls were still visible even though the current deterministic auto-skeleton path did not use them as meaningful generation or evaluation inputs. This made the form noisy and implied unsupported behavior.

## Changes

- Removed the visible ProgramBuilder inputs for program goal, sport:strength ratio, periodization recommendation, excluded memo, excluded exercise selection, preferred exercise selection, and equipment.
- Preserved the active inputs: program name, duration, weekly workout days, daily session minutes, and badminton:strength ratio.
- Kept stored program fields for database and backup compatibility.
- Normalized removed request fields inside the deterministic auto builder so legacy or test request values do not affect generated skeletons.
- Removed the obsolete exercise-selection helper test and added coverage for the cleaned input surface.

## Compatibility

No Room schema or backup format change was made. Deprecated stored fields remain readable and writable internally, but the cleaned ProgramBuilder UI no longer exposes them.
