# v0.4.1.5 Test Hardening Release Notes

v0.4.1.5 is a regression-test hardening release.
It adds focused tests for `CalendarRecordService` and `AnalysisSummaryService` behavior without changing production behavior.

## Changes

- Added `CalendarRecordServiceTest`.
- Added `AnalysisSummaryServiceTest`.
- Enabled JVM unit tests to run Room in-memory database fixtures through Robolectric.
- Bumped app version to `v0.4.1.5` / `401005`.

## CalendarRecordService Coverage

- `copyDate(..., keepConfirmed = true)` preserves mixed confirmed/unconfirmed set state.
- `copyDate(..., keepConfirmed = false)` copies as planned/unconfirmed.
- overwrite copy clears destination records before copying.
- append copy keeps destination records and adds copied records.
- `deleteDateRange(..., includeConfirmed = false)` preserves confirmed sets and reindexes remaining sets.
- `deleteDateRange(..., includeConfirmed = true)` preserves the existing full-delete policy.
- `copyDateRangeAsPlan` copies date offsets as planned/unconfirmed records.
- `moveDate` copies to the target and removes the source.
- `calendarConflictSummary` counts affected dates, existing entries, sets, and confirmed sets.

## AnalysisSummaryService Coverage

- `fatigueAnalysisHistory` returns a stable representative series for confirmed records.
- empty fatigue history input returns the existing safe fallback series.
- `badmintonTransferSummary` uses confirmed transfer records inside the active window.
- `badmintonTransferCoverageSummary` returns the normal data path for transfer records.
- persisted runtime metadata overrides remain reflected in badminton transfer summaries.
- future and out-of-window records do not pollute current summary windows.
