# v0.4.1.10.8 ProgramBuilder Selected Main Exercise Scoring Release Notes

## Start Baseline

- Started from latest main at `v0.4.1.10.7`.
- Baseline commit: `5398433`.
- Release version: `versionName = "v0.4.1.10.8"`, `versionCode = 401018`.

## Problem

- Captain chair leg raise could still be favored repeatedly as a safe filler/core option.
- Main exercises such as squat, deadlift, pull-up, overhead press, and half-kneeling one-arm press needed an explicit final score correction.
- A broad category boost would overcorrect by lifting RDL, rows, bench press, leg press, split squat, other presses, and unrelated core/accessory work.

## Scoring Policy

- Exact selected-main allowlist:
  - `barbell_back_squat`
  - `barbell_deadlift`
  - `pull_up`
  - `ex_32219f7a`
  - `ex_8e1b313e`
- Selected-main multiplier: `1.40`.
- Penalty target:
  - `ex_a345e30b`
  - legacy test key `captain_chair_leg_raise`
- Captain chair leg raise multiplier: `0.70`.
- Application stage: after hard gates, base scoring, and context reranking; before final selection/beam choice.
- Hard gate priority: excluded exercise, safety restriction, and unavailable equipment still win before boost/penalty.
