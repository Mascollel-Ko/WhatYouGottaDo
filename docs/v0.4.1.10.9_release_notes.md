# v0.4.1.10.9 ProgramBuilder Selected Main Slot Repair Release Notes

## Start Baseline

- Started from latest main at `v0.4.1.10.8`.
- Baseline commit: `af286a5`.
- Release version: `versionName = "v0.4.1.10.9"`, `versionCode = 401019`.

## Problem

- v0.4.1.10.8 added a `1.40` score boost for exact selected-main exercises and a `0.70` captain-chair penalty.
- A pure score adjustment still does not protect the path where selected-main candidates are lost before scoring, where the slot plan favors filler/core/transfer work, or where repair leaves a selected-main-missing result.
- The fix needed exact slot reservation and repair, not a broader score tweak.

## Changes

1. `test(program): trace selected main candidate collapse`
   - Added seed-data regression coverage for reservoir survival, score trace visibility, final selected-main presence, captain-chair repetition, and excluded-target priority.

2. `fix(program): reserve exact selected main slots before fillers`
   - Added exact `selectedMainStableKey` slot reservation.
   - Reserved exact selected-main candidates before filler slots for 40+ minute badminton support plans.
   - Kept hard gates authoritative by using only reservoir candidates.

3. `fix(program): block captain chair until selected main requirements are met`
   - Blocked exact captain-chair candidates while selected-main requirements are still open.
   - Kept captain chair possible after selected-main needs are satisfied, capped to one program-wide use.

4. `fix(program): repair filler slots with exact selected main exercises`
   - Added `SELECTED_MAIN_MISSING`.
   - Reopened captain-chair/core/filler/weak slots and inserted exact selected-main reservoir candidates during repair.

5. `fix(program): cap quality scores when selected main exercises are missing`
   - Capped trace-backed selected-main-missing evaluations at 60.
   - Prevented high or perfect scores when exact selected-main candidates were available but absent.

6. `fix(program): expand selected main slot and repair traces`
   - Added selected-main reservation trace keys, captain-chair block trace fields, repair details, and evaluation cap reasons.

7. `chore(release): bump version to v0.4.1.10.9`
   - Bumped app version metadata and finalized release docs.
