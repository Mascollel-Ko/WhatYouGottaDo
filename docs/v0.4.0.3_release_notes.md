# v0.4.0.3 RecordScreen UI Extraction

This release does not add a new user-facing feature.

It extracts `RecordScreen` UI sections into smaller composable files while
keeping record screen state and event wiring in `RecordScreen.kt`.

Key points:
- `RecordScreen.kt` remains the owner of screen-level date, picker, scroll, and ViewModel wiring.
- Record date header and empty state UI are now separate section composables.
- Workout entry card, set row, batch edit dialog, and smash speed card UI are split into focused files.
- Record entry, selected copy, preserve-state selected copy, batch edit, rest timer, metadata, backup, and analysis behavior are unchanged.
- ViewModel, Repository, Room schema, metadata, backup, and analysis public APIs are unchanged.
