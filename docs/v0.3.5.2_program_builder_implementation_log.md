## Pre-push Refinement

- Direct sport sessions are excluded even if malformed metadata marks them selectable. Court footwork, reaction drills, and repeated-effort tests remain eligible according to their own planning metadata.
- Repetition scoring now treats stable anchors separately from accessory, prehab, and transfer rotation.
- Same `stableKey` receives the strongest penalty, `redundancyGroup` receives a soft recency penalty, and weekly `movementFamily` exposure receives a soft budget penalty.
- Session size is capped by prescription-derived work time, inter-set rest, setup time, and a warm-up reserve.
- Validation uses `HARD`, `WARNING`, and `SOFT_PENALTY` severities.
- Added weekly hard-day rules, two-week neuromuscular/joint-impact wave checks, two-week movement coverage, and four-week distribution checks.
- `appCueProfile` remains a candidate/display helper and is not used as a fatigue or analysis axis.

## Compatibility Decisions

- No Room migration: `durationDays` already stores the selected duration.
- Existing program rows remain readable.
- Slot and day-intensity tokens are stored in the existing prescription text.
- `appCueProfile` is candidate-selection metadata only and is never aggregated into analysis.
- Exercise names are display values and explicit exclusion inputs, not classification inputs.
