# v0.4.2.10 Release Notes

v0.4.2.10 activates the approved limited change-of-direction and
rapid-deceleration context modifier for connective-tissue RCV exposure. The
modifier is tissue-specific, bounded, and applied exactly once per derived
event.

- Started from `d5cebbaa6db2e1692c43f9ec7859f3571681ef0c`, where local
  `main`, `origin/main`, and annotated tag `v0.4.2.9` matched.

## Approved Policy

- Runtime formula:
  `E0 = (M / 10) x D_normalized x I_selected x C_resolved`.
- `C_resolved` represents only additional exposure from repeated
  change-of-direction and rapid-deceleration.
- It resolves by exact `exerciseStableKey x loadUnitStableKey`; display-name
  and category heuristics are not used at runtime.
- Approved exercise tiers contain exactly 22 stable keys:
  - `1.09` (9): `ex_bc84eb7f`, `ex_64422511`, `ex_c5f4c242`,

## Load-Unit Boundary

- `cod_context_load_unit_eligibility_v1.csv` reviews all 77 unsided load units
  exactly once.
- Exactly 35 hip, knee, ankle, and foot load units are eligible.
- All pelvic, spinal, shoulder, arm, wrist, and hand load units are explicitly
  neutral.
- `cod_context_modifier_rules_v1.csv` contains the exact 22 x 35 cross product:
  770 rules with strict foreign-key, duplicate, factor, rationale, provenance,
  and policy-version validation.
- No side-specific state or rule was added.

## Runtime Integration

- `TissueContextModifierResolver` returns one factor, status, rule ID, policy
  version, and diagnostic reason for each exercise/load-unit pair.
- `TissueRcvEventLedgerBuilder` resolves C after the final unsided event key is
  known and multiplies it exactly once.
- GROUP/component authority rows still collapse with the existing maximum
  policy and cannot multiply C repeatedly.
- Upper-body and spinal events from a whitelisted sport remain at `1.00`.
- `mappingRoleWeight` remains reserved and exactly `1.0`.
- Authority S and P remain non-multiplicative routing/provenance terms.
- Calculation contract advances to `RCV-EXPOSURE-1.1`; context policy is
  `RCV-COD-CONTEXT-1.0`.

## Rebuild And Compatibility

- Tissue events, summaries, rankings, residuals, and personal 56-day
  distributions are derived on read from original workout records.
- No tissue event or summary cache is persisted, so no Room migration is
  required. Opening analysis rebuilds derived values deterministically under
  the new calculation contract.
- Original workout records, dose inputs, effort selection, PCHIP curves,
  recovery routing, calibration algorithm, OFI, readiness, ProgramBuilder,
  bodyweight volume, daily condition, LOCAL_MUSCLE, UI, and backup formats are
  unchanged.

## Remaining Limitations

- The approved modifier is an explicit whitelist, not a generic COD classifier.
- Pelvic load units remain neutral because the approved scope is lower-limb
  hip/knee/ankle/foot load units.
- `mappingRoleWeight` remains unimplemented pending a separate audit.
- Unrecorded event counts remain missing; this release does not fabricate
  direction-change frequency or duration.
