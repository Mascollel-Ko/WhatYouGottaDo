# v0.3.5.3 ProgramBuilder Architecture

## Purpose

v0.3.5.3 separates template, slot capability, fatigue, exposure, and coverage policy from `ProgramBuilder`. Generated program behavior remains based on the v0.3.5.2 hidden day-count templates. This version does not add the full ratio-band or goal-profile template library.

## Runtime Flow

```text
ProgramSkeletonRequest
  -> ProgramTemplateCatalog
  -> FatigueSlotPolicy
  -> SlotCapabilityResolver
  -> ProgramBuilder scoring and prescription
  -> CoverageAccountingPolicy and existing validators
  -> GeneratedProgramSkeleton
```

`ProgramBuilder` remains responsible for request normalization, candidate filtering/scoring, controlled selection, prescription, item construction, and validator orchestration. It no longer owns day-count templates, week waves, role templates, or fatigue gate construction.

## Components

### ProgramTemplateCatalog

- Owns the current 1-7 day slot skeletons.
- Owns the existing 2-12 week load wave and role ordering.
- Preserves the v0.3.5.2 day, intensity, week type, volume, RPE, and role behavior.
- Exposes centralized `ProgramSlotDefinition` identity for each planned slot.

### ProgramSlotDefinition

Defines the future slot vocabulary and flags: category, anchor, badminton transfer, recovery/prehab, high-impact/reactive, and umbrella status. The new slot vocabulary is representational in this patch and does not force new exercises into generated programs.

### SlotCapabilityResolver

- Reads canonical runtime metadata first.
- Uses structured legacy exercise metadata only when canonical metadata is unavailable.
- Does not classify from display name alone.
- Assigns at most one primary, two secondary, and one weak capability.
- Stores capability source and resolved slot IDs on each generated item for validation and diagnostics.

### ExposureTargetTable

Provides qualitative priorities and non-enforced 2-week/4-week windows. It is a foundation for later template work; v0.3.5.3 does not turn these values into hard quotas.

### FatigueSlotPolicy

Owns the existing `ProgramFatigueGate` and planned-slot intensity adaptation. GREEN, YELLOW, ORANGE, and RED behavior is unchanged: ORANGE softens hard days and RED makes planned days light while current heavy/impact/COD filters remain active.

### CoverageAccountingPolicy

- Primary capability: full credit.
- Secondary capability: partial credit.
- Weak capability: weak credit.
- Umbrella primary capability: partial credit only.
- Existing 2-week movement coverage validation uses these credits first.
- Legacy token fallback remains only for old/test items that do not carry capability data.

## Guardrails Preserved

- Direct sport session exclusion.
- Rehab-like activation cap and repetition penalty.
- Hard-day and rolling neuromuscular/joint-impact checks.
- Two-week movement coverage.
- Four-week distribution and anchor continuity.
- Prescription-based time budget.
- Safe GREEN fallback and ORANGE/RED restrictions.

## Deferred Work

- Ratio-band x goal-profile x day-count template combinations.
- Slot-specific YELLOW/ORANGE/RED substitutions.
- Numeric enforcement of exposure targets.
