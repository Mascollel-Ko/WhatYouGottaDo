# v0.4.1.10.3 ProgramBuilder Refactor Release Notes

## Baseline

- Started from: `v0.4.1.10.2`
- Baseline commit: `dc6d2372dce14798a022a55b4d698b9749856402`
- ProgramBuilder before refactor: 911 lines
- ProgramBuilder after refactor: 286 lines before release commit

## Cause

v0.4.1.10.2 fixed program candidate collapse by widening candidate permissions, preserving fallback slot capability, relaxing required slots, using adaptive pools, and softening optional time-budget trimming.

After that fix, `ProgramBuilder.kt` still owned too many responsibilities directly: candidate domain state, scoring, variety selection, session safety gates, history-based weight suggestion, and selection reason formatting. Keeping those concerns together made future candidate-collapse regressions harder to diagnose.

## Changes

- Added refactor safety tests for candidate collapse behavior.
- Extracted candidate and selection-history domain models.
- Extracted candidate scoring into `ProgramScoringPolicy`.
- Extracted adaptive pool and variety selection into `ProgramVarietyPolicy`.
- Extracted session hard safety gates into `ProgramSessionConstraintPolicy`.
- Extracted history-based weight suggestion into `ProgramWeightSuggestionPolicy`.
- Extracted selection reason string formatting into `ProgramSelectionReasonFormatter`.
- Bumped version to `v0.4.1.10.3` / `401013`.

## Commit Summary

1. `test(program): protect builder refactor behavior`
   - Cause: candidate-collapse protections needed explicit tests before further refactoring.
   - Change: strengthened `ProgramBuilderCandidateCollapseTest` for adaptive pool, required fallback, and budget trimming behavior.
   - Result: refactor safety coverage exists before moving builder internals.

2. `refactor(program): extract candidate domain model`
   - Cause: candidate identity, fit helpers, selection history, and history weight state were embedded in ProgramBuilder.
   - Change: added `ProgramCandidate.kt`, `ProgramSelectionHistory.kt`, and initial history weight helper extraction.
   - Result: candidate state became reusable and inspectable outside the builder.

3. `refactor(program): extract scoring policy`
   - Cause: hard gates and soft score preferences were too close in ProgramBuilder.
   - Change: added `ProgramScoringPolicy.kt` for candidate ranking and exposure balance adjustment.
   - Result: scoring remains ranking-only and does not decide eligibility.

4. `refactor(program): extract variety selection policy`
   - Cause: adaptive pool sizing and controlled candidate selection were still inside ProgramBuilder.
   - Change: added `ProgramVarietyPolicy.kt`.
   - Result: fixed top-3 style candidate narrowing stays isolated from orchestration and should not return.

5. `refactor(program): extract session constraint policy`
   - Cause: hard safety gates for session composition were mixed with candidate orchestration.
   - Change: added `ProgramSessionConstraintPolicy.kt`.
   - Result: rehab activation caps, isolation caps, heavy lower repeat limits, and heavy lower/impact/COD blocking are grouped as session hard gates.

6. `refactor(program): extract weight and reason helpers`
   - Cause: history-based weight suggestions and reason text formatting were helper responsibilities inside ProgramBuilder.
   - Change: added `ProgramWeightSuggestionPolicy.kt` and `ProgramSelectionReasonFormatter.kt`.
   - Result: ProgramBuilder delegates recommendation and explanation helpers.

7. `chore(release): bump version to v0.4.1.10.3`
   - Cause: the refactor needed release metadata and durable worklog documentation.
   - Change: bumped version and added this release note.
   - Result: v0.4.1.10.3 records the refactor scope and verification path.

## File Responsibility Map

- `ProgramBuilder.kt`: orchestrates request normalization, template scheduling, slot query calls, policy delegation, generated item assembly, validation issue rendering, periodization choice, and default name generation.
- `ProgramCandidate.kt`: candidate identity, metadata-derived fit helpers, eligibility helpers, and candidate trace/result types.
- `ProgramSelectionHistory.kt`: selected stable keys, redundancy/family exposure, repeat checks, and exposure accounting state.
- `ProgramScoringPolicy.kt`: candidate ranking score and exposure balance adjustment.
- `ProgramVarietyPolicy.kt`: adaptive pool size, controlled selection, repeat allowance delegation, and selection history recording.
- `ProgramSessionConstraintPolicy.kt`: session-level hard safety gates.
- `ProgramWeightSuggestionPolicy.kt`: confirmed-history based weight suggestion and weight source labels.
- `ProgramSelectionReasonFormatter.kt`: selection reason string formatting only.
- `ProgramCandidateInventory.kt`: active/selectable/equipment/excluded candidate inventory.
- `ProgramSlotCandidateQuery.kt`: slot-stage filtering, fallback tiering, candidate trace counts, and required-slot relaxation.
- `ProgramPrescriptionPolicy.kt`: sets/reps/seconds/RPE prescription and time-budget fitting.

## Behavior Preserved

- Candidate trace count stages remain available on generated skeletons.
- Required slot relaxation remains active for exact/partial/role/family/weak/legacy/name fallback.
- Adaptive selection pools remain wider than fixed top-3.
- Optional time budget trimming still reduces optional prescription before skipping only that slot.
- Candidate-collapse safeguards remain covered by focused tests.
- No intentional ProgramBuilder generation behavior change was introduced in this release.

## ProgramBuilder Remaining Responsibilities

- Normalize the request.
- Build the selected template schedule.
- Coordinate candidate inventory/query/scoring/variety/session/prescription policies.
- Assemble `ProgramSkeletonItem` values.
- Render validation warnings and issues.
- Choose periodization and default program name.
