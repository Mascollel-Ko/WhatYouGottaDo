### 5.1 `FatiguePresentationSnapshot`

목적:

```text
Future Home summary, Analysis fatigue UI, Today Readiness display, and ProgramBuilder preparation can read one shared presentation snapshot instead of each recalculating fatigue differently.
```

Current shape:

```kotlin

    val overallScore: Int,
```

Field meaning:

| Field | Meaning |
|---|---|
| `overallScore` | Public 0..100 overall fatigue score. |

| `jointTendonScore` | Public joint/tendon/impact score from deceleration, SSC, overhead, grip categories plus supporting body parts. |
| `systemicScore` | Public systemic fatigue score from `SYSTEMIC`, with limited `BADMINTON_COURT` support. |
| `focusScore` | Public focus/court-cognitive proxy score from `NEURAL_SPEED` and `BADMINTON_COURT`. |
| `highCategories` | High internal category pressures for debugging/UI explanation. |
| `highBodyParts` | High body-part pressures for debugging/UI explanation. |
| `gate` | Future ProgramBuilder-oriented gate snapshot. |
| `reduceToday` | UI-ready list of training targets to reduce today. |
| `availableToday` | UI-ready list of relatively available training targets. |

## 7. Score Normalization

The mapper converts each `FatiguePressure` into a public `0..100` score.

### 7.1 Raw Inputs Used

The score function uses these existing fields:

```kotlin
FatiguePressure.currentResidualLoad
FatiguePressure.pressure
FatiguePressure.zScore
FatiguePressure.percentile
FatiguePressure.level
```

It does not invent new fatigue metrics.

### 7.2 Zero and Missing Safety

Missing pressure item:

```text
score = 0
```

Zero or near-zero residual load:

```text
score = 0
```

Current guard:

```kotlin
if (item == null || item.currentResidualLoad <= TodayReadinessConstants.LOW_STD_FLOOR) return 0
```

### 7.3 Raw Score Candidates

Current score candidates:

```kotlin
item.pressure?.let { pressure -> (pressure.coerceIn(0.0, 2.0) * 50.0).roundToInt() }
item.zScore?.let { zScore -> (zScore.coerceIn(0.0, 2.5) / 2.5 * 100.0).roundToInt() }
item.percentile?.roundToInt()
levelFloor(item.level)
```

Then:

```text
rawScore = max(candidate scores)
```

Rationale:

```text
If any pressure signal indicates high fatigue, the public score should reflect that.
This matches the existing TodayReadinessDecisionEngine style, which reacts to high levels rather than averaging them away.
```

### 7.4 Level Floors and Caps

The mapper uses `FatigueLevel` to keep score ranges aligned with existing readiness levels.

Current floors:

| FatigueLevel | Floor |
|---|---:|
| `LOW` | 0 |
| `NORMAL` | 35 |
| `ELEVATED` | 60 |
| `HIGH` | 75 |
| `VERY_HIGH` | 90 |
| `LIMITED` | 100 |

Current caps:

| FatigueLevel | Cap |
|---|---:|
| `LOW` | 30 |
| `NORMAL` | 59 |
| `ELEVATED` | 69 |
| `HIGH` | 89 |
| `VERY_HIGH` | 100 |
| `LIMITED` | 100 |

This means:

```text
HIGH pressure becomes at least 75.
ELEVATED pressure stays below restricted threshold.
VERY_HIGH pressure can reach 100.
```

### 7.5 Final Clamp

Every public score is clamped:

```kotlin
private fun Int.clampScore(): Int = coerceIn(0, 100)
```

All public scores must stay in:

```text
0..100
```

### 9.6 `overallScore`

Derived from:

```text
neuralScore
localMuscleScore
jointTendonScore
systemicScore
focusScore
```

Current code:

```kotlin
val axisScores = listOf(neuralScore, localMuscleScore, jointTendonScore, systemicScore, focusScore)
val meanScore = axisScores.average()
val maxScore = axisScores.maxOrNull() ?: 0
val highAxisBoost = axisScores.count { score -> score >= RESTRICTED_SCORE } * 4 +
    axisScores.count { score -> score >= VERY_HIGH_SCORE } * 3
val overallScore = (meanScore * 0.45 + maxScore * 0.45 + highAxisBoost).roundToInt().clampScore()
```

Heuristic:

```text
overallScore = mean * 0.45 + max * 0.45 + high-axis boost
```

The code comment marks this as beta heuristic:

```text
These weights are transparent tuning defaults, not research-derived constants.
```

## 10. Axis Combination Heuristic

Function:

```kotlin
private fun combineAxisScores(vararg scores: Int): Int
```

Current behavior:

```text
1. Drop zero scores.
2. If no positive score, return 0.
3. Take the max score as dominant.
4. Add 20% of the supporting mean.
5. Clamp to 0..100.
```

Current code:

```kotlin
val max = positiveScores.maxOrNull() ?: 0
val supportingMean = positiveScores
    .filter { score -> score != max }
    .takeIf { values -> values.isNotEmpty() }
    ?.average()
    ?: 0.0
return (max + supportingMean * SUPPORTING_AXIS_WEIGHT).roundToInt().clampScore()
```

Current support weight:

```kotlin
SUPPORTING_AXIS_WEIGHT = 0.20
```

Reason:

```text
The highest pressure axis should dominate.
Supporting pressure should matter, but should not double count the same training session too heavily.
```

## 12. Volume Factor and RPE Cap

### 12.1 `volumeFactor`

Current mapping:

| Overall score | Volume factor |
|---:|---:|
| `>= 75` | `0.25` |
| `>= 60` | `0.50` |
| `>= 45` | `0.75` |
| `< 45` | `1.00` |

Current code:

```kotlin
private fun volumeFactor(overallScore: Int): Double =
    when {
        overallScore >= 75 -> 0.25
        overallScore >= 60 -> 0.50
        overallScore >= 45 -> 0.75
        else -> 1.0
    }
```

### 12.2 `rpeCap`

Current mapping:

| Condition | RPE cap |
|---|---:|
| `neuralScore >= 70` | `7` |
| `neuralHeavyScore >= 70` | `7` |
| `overallScore >= 60` | `7` |
| `overallScore >= 45` | `8` |
| otherwise | `null` |

Current code:

```kotlin
private fun rpeCap(overallScore: Int, neuralScore: Int, neuralHeavyScore: Int): Int? =
    when {
        neuralScore >= RESTRICTED_SCORE || neuralHeavyScore >= RESTRICTED_SCORE || overallScore >= 60 -> 7
        overallScore >= 45 -> 8
        else -> null
    }
```

Meaning:

```text
High neural or high overall fatigue should cap intensity.
Moderate overall fatigue gets a softer cap.
Low fatigue has no cap.
```

This is prepared for future ProgramBuilder usage, but ProgramBuilder is not wired yet.
