# v0.4.2.15 Release Notes

v0.4.2.15 corrects the OFI five-axis regression. OFI now uses exactly `고중량·힘 신경계`, `전신 근육`, `국소 근육`, `고속`, and `반응`, while connective-tissue load and recovery remain a separate 77-unit analysis.

## Baseline

- Started from latest `origin/main` at `dbad9fe02d1a6e709c59c65149ba5ce7c9be7bf5`.
- Previous app identity: `v0.4.2.14 / 402014`.
- OFI implementation commit: `aa08b49ff183c60c45c9e8bf95a9542df1b592ce`.

## Cause

The active OFI model and governed documents had incorrectly made joint/tendon/impact and movement/focus the fourth and fifth OFI axes. That contradicted the separate connective-tissue protocol and coupled workload fatigue presentation to legacy axis semantics.

## Changes

- Replaced the legacy OFI state, classifier targets, warnings, Home summary, Analysis summary and five-card labels with the canonical five axes.
- Removed `JOINT_TENDON_CAUTION` and the legacy movement/focus warning path from OFI.
- `HIGH_SPEED` now uses workload plus movement family/subtype, program slot, primary/secondary stress, sport context, badminton transfer, skill-target and physical-quality signals for speed, sprint, plyometric/SSC, acceleration, first-step, court movement and footwork.
- `REACTIVE` now independently uses reaction, decision, visual-tracking, direction-change, random/beep, reactive-agility, footwork, agility, shuttle and court-movement signals.
- Representative duration/RPE badminton records contribute to systemic, high-speed and reactive load. The high-speed and reactive formulas are independently testable and do not reuse legacy axis values.
- Discomfort-driven joint/tendon coaching remains available, but it is no longer triggered by an OFI axis.
- Updated the existing OFI core, classification, five axis documents, protocol registry and protocol index to protocol version `1.1.0`; no duplicate protocol was created.

## Preserved

- The connective-tissue mappings, 77 load units, exposure formulas, recovery curves, priors, `PersonalBaseline`, `w_perUnit`, state classification and education copy are unchanged.
- The tissue numerical regression remains raw `100.0` and adjusted `120.0` for the fixed reference fixture.
- Strength and badminton volume protocols, ProgramBuilder and exercise stable keys are unchanged.
- Runtime metadata storage fields retained for compatibility are not OFI axes and do not directly drive `HIGH_SPEED` or `REACTIVE`.
