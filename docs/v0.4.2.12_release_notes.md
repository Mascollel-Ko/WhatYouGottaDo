# v0.4.2.12 Release Notes

v0.4.2.12 activates the validated connective-tissue prior registry and adds per-unit personal baselines. It replaces the calibration-only connective-tissue result with relative states while preserving the production exposure, recovery, symptom override, OFI and unrelated feature behavior.

## Baseline

- Started from latest `origin/main` at `d83ae6441642fb5f8f7f41dc332d9c7ad9a225ad`.
- Previous app identity: `v0.4.2.11 / 402011`.
- Release identity: `v0.4.2.12 / 402012`.
- Existing user-owned `outputs/*` modifications were not touched or staged.

## Cause

The Phase 1 prior registry was generated and validated but runtime still gated the main connective-tissue result behind one global 56-observation-day percentile history. That withheld a useful result from new users, treated elapsed wall-clock history too broadly, and could not express unit-specific exposure sufficiency or stale-history gaps.

## Changes

### Calendar history and per-unit weights

- The anchor is the later local date of the last confirmed workout and the last explicitly persisted DailyCheckIn.
- Exactly A-6 through A are excluded from personal calibration; A-7 is eligible.
- Global no-workout and unit no-exposure gaps use explicit chronological segments and exact 6/7/13/14/27/28 boundaries.
- The same global/unit boundary applies the stricter retention once; separate boundaries accumulate.
- `w_span = clamp(weightedValidObservationDays / 56, 0, 1)`.
- `w_exposure = clamp((weightedDistinctExposureDays - 3) / 9, 0, 1)`.
- `w_perUnit = min(w_span, w_exposure)` and same-day activity counts once per unit.

### Personal baseline and prior activation

- Historical samples reconstruct production residual load at the current local hour with no future-event leakage and DST-safe local dates.
- Personal Q30/Q80/Q95 use deterministic cumulative weighted nearest-rank over samples strictly above `meaningfulFloor`.
- Invalid personal distributions fall back to the exact adjusted prior with diagnostics.
- Runtime parses the app-ready generated registry by canonical stable key and local hour; no display-name matching or in-app simulation exists.
- Final Q30/Q80/Q95 linearly mix adjusted prior and personal boundaries with one effective weight.
- `CurrentLoad` remains the production residual value and is never multiplied by profile adjustment or calibration weight.

### Profile and compatibility

- Reuses canonical bodyweight, existing strength-training years and separate badminton/racket-sport years.
- Adds nullable persistent habitual training intensity: `LIGHT/NORMAL/HARD` (`가벼운 편/보통/강한 편`). It is not inferred from session RPE, recent volume, OFI or residual load.
- Adds Room migration 20→21 and schema export.
- Backup export/import supports the new value, old backups, missing values and unknown values with a neutral null fallback.

### Relative states and UI

- Exact labels are `낮은 편`, `평소 범위`, `높은 편`, `매우 높은 편`, and genuine authority failure `판단 불가`.
- Short personal history is not `보정 중` and is not unavailable when a valid prior exists.
- Removes the old displayed percentile and `56일 보정 후 표시` placeholder.
- Keeps relative state primary and adds one low-emphasis footer at the bottom:
  - `기준 출처: 조직별 초기 기준`
  - `기준 출처: 개인 운동 기록`
  - `기준 출처: 조직별 초기 기준·개인 운동 기록 혼합`
  - Mixed only: `조직마다 개인 기록의 반영 정도가 다를 수 있습니다.`
- Keeps top-three expansion, educational dialogs, contributors, scrolling and symptom override behavior.

## Limitations

- The generated prior is a representative product prior, not a population norm.
- Gap retention, calibration weights, scenario weights, shared profiles and experience coefficients are product policy, not measured biological adaptation.
- Relative states are not diagnosis, injury probability, tissue damage, tissue capacity, a safe-load guarantee or an exact recovery clock.
- Some recovery and mapping inputs remain class-level or low-confidence proxies.
