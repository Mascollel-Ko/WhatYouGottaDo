# v0.4.0.0 Stabilization Baseline

This release consolidates the late v0.3.9.x stabilization work.

Included baseline fixes:

- Preserve-state selected copy work from the v0.3.9.8 line.
- Exercise metadata override resolution before analysis.
- Muscle key normalization for load analysis, including quadriceps override cases.
- Muscle trend legend improvement.
- Release workflow note: future work must start from latest `origin/main`, not from old release tags.

Known technical debt:

- `TrainingRepository` remains too large and should be split in a future refactor.
- `RecordScreen` and `PlanScreen` remain large Compose screens and should be decomposed later.
- Analysis/data paths need continued regression tests around resolved metadata and canonical keys.

This is not a large feature rewrite. It is a stable baseline before future refactoring.
